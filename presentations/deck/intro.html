Look at these screenshots from System Dynamics modelling tools...

Clockwise from top-left: Stella, Simile, Insight Maker, Sysdea, and Vensim in the middle.

We see that these are

similar...

They all can display the model diagram, plot of results, input sliders etc.

... but different

Different screen layout, different appearance of the diagrams, etc.

This is hugely wasteful - massive duplication of effort.

It's also restrictive - any one package has to limit the number of features it provides.

For the user, it also leads to lock-in - you get used to one package.

Car or kitchen?

When you choose a car...

... you are choosing a whole package of (good and not so good) features.

You choice is a compromise between price, performance, styling, reliability, fuel economy, etc

Just about all software is like a car.

When you choose a kitchen...

... you choose each component separately, and combine them together to make the kitchen you want.

This works because there is a standard infrastructure (electricity, gas, water).

Systo is like a kitchen - or, to be precise, Systo is like the concept of kitchen.

Systo features

  • Multiple diagrammatic languages
  • Large number of diagramming, input and simulation display widgets
  • Highly efficient simulators
  • Built-in interactive tutorials
  • Unlimited flexibility in designing user interfaces, using HTML and JavaScript
  • Models, language definitions and widgets can live anywhere on the web
